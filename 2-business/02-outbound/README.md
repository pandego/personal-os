# Outbound

Outbound systems for getting conversations, opportunities, and replies without scattering drafts across the repo.

## Purpose
Use this folder for proactive outreach and demand-generation workflows.

This is the place for:
- newsletter planning and copy
- LinkedIn outreach messages and follow-up sequences
- Upwork proposals and reusable proposal building blocks

## Structure
- `01-newsletters/` - newsletter drafts, campaign notes, themes, and experiments
- `02-linkedin-outreach/` - connection messages, follow-ups, sequence ideas, outreach notes
- `03-upwork/` - proposal drafts, answer banks, positioning snippets, job-fit notes

## How to Use Each Folder
### `01-newsletters/`
Use this when the goal is one-to-many communication.

Good examples:
- newsletter ideas worth drafting later
- issue themes and hooks
- campaign sequences tied to launches or offers
- post-send notes on what performed well

### `02-linkedin-outreach/`
Use this for one-to-one or small-batch outreach on LinkedIn.

Good examples:
- first-touch messages
- follow-up sequences
- industry-specific variations
- notes about what gets replies vs what gets ignored

### `03-upwork/`
Use this for Upwork opportunity workflows.

Good examples:
- reusable proposal skeletons
- answer banks for common client questions
- strong opening paragraphs
- notes about which jobs are a fit and why

## Operating Rules
- Keep outreach human - no generic spam sludge.
- Save patterns that work, not just every message ever written.
- Separate reusable templates from opportunity-specific drafts.
- If a message is tied to a real client, move delivery work to `2-business/03-clients/` once it becomes active work.

## Lightweight Suggested Conventions
- one file per draft or reusable pattern
- use descriptive names instead of vague scratchpad names
- when useful, prefix dated drafts with `YYYY-MM-DD-`
- keep final/reusable patterns easy to find and easy to adapt

## Quality Bar
Before considering an outbound draft ready, check:
- Is the audience clear?
- Is the ask small and easy to respond to?
- Does it sound like a real person instead of a funnel robot?
- Is there a specific reason this message exists now?
- Could future-you reuse it without re-decoding the context?
